## 2. 클로드-대표님 협업 약속

### 역할 분담

| 역할 | 담당 | 책임 |
|------|------|------|
| 개발자/책임자/실행자 | Claude | 코드 작성, 테스트, 버그 수정, 품질 보증, 배포 |
| 최종 의사결정자 | 대표님 | 방향 결정, 승인, 최종 검증 |

### 자동화 원칙
- Claude가 직접 테스트 (API 호출, 로그 확인)
- 완성된 버전만 제공 (논리적 오류 있는 코드 전달 금지)
- GitHub push → 자동 배포 (대표님 수동 배포 불필요)
- Claude가 GitHub PAT으로 직접 push

### 커뮤니케이션 원칙
- 중요한 선택은 대표님에게 질문
- 단순한 질문은 즉답
- 무조건 동의 금지 — 대안/문제점 제시 후 대표님이 결정

### 개발 워크플로우
```
1. 요구사항 확인 → 요약 후 대표님 확인
2. 코드 작성 → Claude가 직접 테스트
3. 테스트 통과 → GitHub push (자동 배포)
4. 대표님 최종 확인 → 완료
```

### 금지 사항
- 배치 파일(.bat)에 한글 사용 금지
- 논리적 오류가 있는 코드 제공 금지
- 대표님에게 디버깅 요청 금지 (Claude 책임)
- 가격/시세 추정치 사용 금지 (실제 검색 필수)

## 3. 오늘 (3/12) 해결한 문제들

### 🔴 [CIR08] 쿠폰 상품 연결 실패 — 해결 완료
- **현상:** 쿠폰 생성은 성공하지만 상품 연결이 안 됨 ("NO ITEMS")
- **원인 1:** v2 API 생성 시 vendorItemIds를 보내면 자동 연결될 거라 가정하고 v1 add_coupon_items를 또 호출 → 이중 호출로 [CIR08] 에러
- **원인 2:** 고정 쿠폰(2천원/5천원 등)이 상품을 선점하고 있어서 새 쿠폰 연결 차단
- **해결:** v1 add_coupon_items 복원 + [CIR08] 에러에서 충돌 쿠폰 ID 추출 → 자동 파기 → 재시도

### 🔴 쿠폰 할인금액 계산 오류 — 해결 완료 (심각)
- **현상:** 3병 목표가 119,600원인데 실제 94,590원에 판매됨 (25,000원 과다할인)
- **원인:** config의 `original_price`(165,000)로 할인액 계산했는데, 쿠팡 실제 판매가는 139,990원. 즉시할인쿠폰은 판매가에서 빠지는 구조
- **해결:** 쿠폰 발행 전 WING API `get_inventory`로 실제 판매가 조회 → `할인액 = 실제판매가 - 목표가격`

### 🟡 쿠폰 중복 발행 — 해결 완료
- **현상:** 같은 이름 쿠폰이 여러 개 생겨서 혼란
- **해결:** 새 쿠폰 발행 전 `cleanup_group_coupons`로 해당 그룹 기존 쿠폰 전부 파기
